# Modality landscape — multi-disease cross-tab

**Status**: case-study landscape — a multi-disease cross-tab of the
case-study portfolios currently on disk against the 24-axis surface
from `AXIS/HIERARCHY.tape`. The structured cross-tab artefact
(`LANDSCAPE.md`) is **auto-generated by `landscape_generator.py`**;
this `README.md` is the hand-authored introduction.

## §0 Honest scope fence

This is a **multi-disease pilot cross-tab**, NOT the deferred
200-disease re-mapping that `AXIS/HIERARCHY.tape` Log flags as
out-of-scope work. The cohort here is whatever
`case_studies/<name>_portfolio/` directories actually exist at
generation time — five today (BCL2, COVID-19, HIV-1, KRAS-G12C, SMA),
each one is its own one-disease pilot writeup (see §5 for cross-links
to each portfolio's `README.md`).

The cross-tab is structured discipline, not a recommendation:

- **NOT** a therapeutic, clinical, oncology, antiviral, regulatory,
  efficacy, or portfolio-recommendation claim. Every cell that
  carries a drug name is read directly from that portfolio runner's
  own witness JSON (`in_scope_drugs[].drug` or `drug_name` —
  governance **g1**). No drug is invented; no axis mapping is
  invented.
- **NOT** a ranking. The three honest cell categories (FDA in-scope ·
  research-stage · UNPLACED) are independent dimensions reported
  side-by-side — same reporter ≠ ranker fence as
  `selftest/portfolio_fitness_function.py` and the G4 PEPTIDE ×
  MACROCYCLE cross's `comparison_is_ranking = false` schema const.
- **NOT** a comprehensive map. Most cells are honestly empty (`·`) —
  most disease × axis combinations have no FDA drug AND no
  research-stage candidate, and that is the truth. The landscape is
  HONESTLY SPARSE.
- **NOT** lattice-derived (governance **f_lattice_fit**). The 24-axis
  surface is the curator-fixed core-5 + expansion-main + sub-axis
  hierarchy from `AXIS.tape` + `AXIS/HIERARCHY.tape`. None of the
  cell content is derived from the n=6 lattice.
- **NOT** a CBER-scope expansion. UNPLACED rows (CBER /
  CDRH-regulated biologics from a portfolio's `not_in_scope_drugs[]`
  or from research-stage negatives flagged
  `fda_center_if_filed='CBER'`) do NOT bind to any axis column. They
  are reported per-disease — listing them keeps the criterion-#4
  drug-only / CDER discipline honest without inventing axis coverage
  the repo does not have.

Every cell observation is **in-silico simulator-consistency only**
(governance **g8 / f2**). The cross-tab is a project-state REPORT.

## §1 The cross-tab

The generated cross-tab lives in `LANDSCAPE.md` (overwritten on each
run). To regenerate:

```bash
python3 case_studies/landscape/landscape_generator.py
# expected: exit 0, last line  __LANDSCAPE_GENERATOR__ PASS
# (or __LANDSCAPE_GENERATOR__ SKIP honestly if no portfolios exist)
```

Cell glyph legend:

| glyph | meaning |
|:---|:---|
| **`✓ <drug-name>`** | FDA-approved IN-SCOPE drug on this axis. The drug name is read verbatim from the portfolio runner's `in_scope_drugs[].drug` (or `drug_name`) field. Two drugs on the same (disease, axis) cell are joined with `+` (e.g. KRAS-G12C × COVALENT → `✓ sotorasib + adagrasib`). |
| **`🔬 research`** | A research-stage candidate is listed for this axis in the portfolio's `research_stage_negatives[]` (with `axis_in_repo` recognised), OR a clinical-stage drug is listed in `in_scope_drugs[]` with `fda_approval_status='clinical_stage'` (e.g. BCL2 × PPI is downgraded for navitoclax). Honesty: this is NOT a FDA drug — it is research / clinical-stage modality coverage. |
| **`⚠ UNPLACED`** | A CBER / CDRH scope-disqualified candidate is listed at the disease level. By design, UNPLACED rows do NOT bind to any axis column — they appear only in the §4 per-disease honesty notes. (Implementing a CBER-scope code axis would breach **g8** + criterion #4 — the same pattern that already governs GENETIC-MEDICINE / ADC / THERANOSTIC.) |
| **`·`** | No candidate / not applicable. Most cells carry this glyph — the cross-tab is honestly sparse. |

## §2 Per-disease + per-axis totals

`LANDSCAPE.md` carries two stable summary blocks underneath the
cross-tab:

- **§2 per-disease totals** — for each disease: the number of FDA
  in-scope drugs (raw count from `in_scope_drugs[]`, NOT distinct
  axes — a portfolio may map several drugs onto one axis), the
  number of research-stage cell contributions (every per-axis hit,
  with multiplicity), and the count of disease-level UNPLACED notes.
- **§3 per-axis totals** — for each of the 24 axes: how many cells
  in the cohort carry an FDA-approved drug, and how many carry a
  research-stage signal. The vast majority of axes have **zero FDA
  cells in this cohort** — this is the truth (most axes have no
  FDA-precedent disease in the small five-disease cohort, and the
  deferred 200-disease re-mapping is exactly what would fill more
  of them).

## §3 Honest empty-cell discussion

The cohort is `R × C` cells where `R` = number of portfolio
directories on disk and `C = 24` axes. With today's five-disease
cohort that is `5 × 24 = 120` cells. Of these only a small
fraction carry an FDA name; most are `·`. The empty-cell majority
is **not a gap** — it is the truth-telling cell. A landscape that
filled every cell would have to invent drug-precedent or
research-precedent that does not actually exist; that would breach
**g3** (honesty-external) and **f1** (lattice-fit-on-external-entity)
and **f2** (wet-lab-clinical-claim-from-in-silico).

What the sparsity is, plainly:

- Most modalities have a small pool of FDA precedents and a small
  pool of disease applications. Five diseases × twenty-four
  modality-categories will be sparse for the same reason any
  small-finite × small-finite cartesian projection is sparse.
- The expanding-but-research-stage modalities (PROTAC / LYTAC /
  AUTAC / RIBOTAC / COVALENT-DEGRADER / MOLECULAR-GLUE / AUTAC /
  APTAMER / METALLODRUG-for-non-cisplatin-indications / etc.) carry
  a small set of named research candidates per axis in
  `AXIS/HIERARCHY.tape`'s @D entries — but those candidates have NOT
  reached FDA approval against the diseases in this cohort. The
  empty `·` cell records that absence honestly; an
  attribute-anything-to-something cell would not.
- UNPLACED (CBER / CDRH) rows do NOT bind to axis columns by design.
  The repo's drug-only / CDER scope (criterion #4) is a deliberate
  fence; the landscape inherits it.

## §4 Governance

- **g1 real-limits-first**: every cell value is read from a runner's
  own witness JSON. The generator has no synthetic 'expected drug'
  table; nothing is invented to fill a cell.
- **g3 honesty-obligation-external / f1 lattice-fit-on-external-
  entity**: drugs are described only by their own published
  precedent (the witnesses already carry those — the landscape
  re-uses the witness data verbatim).
- **g7 skip-is-honest**: if no portfolios exist on disk the
  generator emits a SKIP sentinel and exits 0 — discovery failure is
  honest, not a fatal error. If a runner times out / exits non-zero
  the row is recorded SKIP but the gate still PASSes provided at
  least one portfolio scored.
- **g8 in-silico-only / f2**: the cross-tab characterizes the
  in-silico portfolio-cohort shape only. NEVER a therapeutic /
  clinical / regulatory / efficacy / portfolio-recommendation claim.
- **f_lattice_fit**: the 24-axis surface is the curator-fixed
  hierarchy from `AXIS/HIERARCHY.tape`; no axis count, no axis
  position, no cell rule is derived from the n=6 lattice.
